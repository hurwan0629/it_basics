# 네트워크 형태
네트워크 구성을 알아보기 위해 한번 정리를 해보려한다.
- 인터넷
- 사설 네트워크 (이더넷) 
으로 나누어서 정리하겠습니다.
## 인터넷
인터넷은 ISP(인터넷 서비스 제공업체)로 부터 시작이 됩니다. 전파를 통한 무선도 가능하짐나 일단 유선 기준으로 말해보겠습니다. 각 회사로부터 뻗어나가는 회선(전용 또는 일반)을 통해 내려갑니다. 이는 ISP들의 ONU(Optical Network Unit)들에 도착하게 됩니다. 이는 다중 사용자를 지원하여 여러 사용자가 한번에 사용이 가능하며 
- ONU 패시브: 전원이 필요 없어 하나의 광섬유를 여러 가입자가 공유합니다.
- ONU 액티브: 능동 소자(스위칭 장비, 리피터) 등을 사용하여 신호를 관리합ㄴ디ㅏ. 이를 통해 각 가입자가 전용 대역폭을 가집니다. 데이터센터 등에 쓰입니다.

이를 통헤 **엣지 라우터**를 통해 보통 공유기 드릉ㄹ 통해 전달이 됩ㄴ디ㅏ.

이후에는 방화벽(3,4계층)을 통해 미리 설정된 방식, 포트, ip등을 통해 1차로 거릅니다.

들어온 트래픽은 IPS와 IDS를 통해 내부적으로 차단/탐지(경고)를 하게 됩니다.

이후에는 WAF 등을 통해 7계층 트래픽 검사를 해줍니다. 이후 로드밸런싱을 통해 WAS 또는 WebServer 과 ㅓㅈㅂ하게 됩니다.

## 사설 네트워크 (회사같은)
ISP회선 안에서 엣지 라우터를 하나 설치한다.
그 뒤에 방화벽으로 3,4계층 트래픽을 차단하여 외부로부터의 접근을 통제해주어 사설 망을 성립시킨다. 이후에 스위치를 통해 네트워크를 나누거나(VLAN), 서로 통신이 가능하게 만들어준다.
이렇게 만들어진 공간을 기반으로 DHCP, DNS를 통해 IP할당 및 탐지를 해주게 하며, ServerZone에서 여러 공용 자원을 사용 가능하게 해준다(직원들이) 이후에는 2스위치 안에 AccessSwitch를 하나 더 추가하여 호스트 (직원PC, 프린터, 등등)를 등록하여 사용한다.

# ISP
ISP는 KT, LG같은 인터넷 제공업체를 말하며 
ISP 백본망은 지역 통신국 또는 군사에서 시작되어 동네쪽 광케이블로 가며, 회사 건물 회선 종단장비 -> 회사 라우터/방화벽으로 뻗어나가게 된다.

예를 들어서 외국의 데이터센터 -> 경기도의 호스트에 5Gb를 받는데에도 몇분이면 충분한데 이게 진짜로 내 컴퓨터 -> 카페 라우터 -> 광케이블 -> 지역 통신국 -> ISP 망 -> 한국 밖 광케이블 식으로 이어져서 "파일 주세요" 한다음에 5Gb 파일이 오게된다.

이때 40Gb를 요청하게 된다면 만약 1000BASE-LX가 사용된다? 라면 대략 40초가 걸린다. (혼잡제어, TCP질의, 처리 속도를 제외하면)

사설 네트워크 하면 주요 구성 요소는
- 라우터(게이트웨이)
- 방화벽
- 스위치 (L2: MAC를 통한 전달, L3: IP를 통한 전달, 코어, 엑세스)
  - L2는 MAC를 통해서 통신을 하며 같은 VLAN 안에 있는 기기들끼리는 바로바로 통신이 가능하다.  (보안은 포트, MAC, VLAN, ARP, 등정도를 방어해준다.)
  - L3는 다른 VLAN끼리의 통신이 가능하게 해준다. IP를 보고 라우팅을 해주는 역할을 해준다. 
- VLAN
- DHCP
- DNS
- AD(LDAP)
- IPS, IDS
- Proxy, ReverseProxy
- WAF 
- NAC
- 서버존 (WS, WAS, DB, FILE, BACKUP, LOG, Monitoring)
- 관리망
- 보안 소프트웨어 (장비 안, 별도 노드, 에이전트)

# 나의 설명 - 같은 VLAN 안 통신
L3에 존재하는 여러 VLAN 중에서 192.168.10.0/24가 존재하는데 여기에 두 PC 1,2 가 있다.

이떄 PC1이 PC2에 데이터를 보내려 하면 PC1은 자신이 연결된 네트워크 인터페이스중에서 외부로 나갈때 쓰는 인터페이스를 선택하여 통신을 보낸다. (만약에 MAC위치를 알 수 있다면 즉시 보낸다.) 이때 PC2의 위치를 모른다면 L2스위치에 "브로드캐스트 해줘. PC2(IP)찾게." -> L2스위치가 브로드 캐스트 -> PC2가 허용된 요청임에 따라 자신의 MAC를 알려준다. "나야 PC2" -> L2를 통해 다시 PC1로 MAC 주소가 간다. -> PC1이 받은 MAC로 다시 통신을 보내면 L2를 거쳐 PC2로 가게 된다. 이때 L2는 [포트번호:MAC] 테이블이 있어 적절히 신호를 애니캐스팅 가능하다.

### 고칠부분
L3는 여러 VLAN이 아니라 여러 VLAN의 게이트웨이를 가지고 있다.
VLAN은 네트워크 대역이며 그 대역의 게이트웨이를 L3 스위치가 가지고 있다.

L2스위치가 대신 브로드 캐스트 해주는거보다는 **브로드캐스트 프레임**을 PC1이 보내서 L2의 VLAN 전체에 뿌려주는 방식이다. 

애니캐스팅-여러 후보중 가까운 하나에게 보내는 것
**유니캐스트**가 맞음


# 나의 설명 - 다른 VLAN 기본 (스위치)
VLAN은 하나의 스위치의 포트들을 그룹으로 나누는 것이다. 이는 네트워크 대역폭으로 나눠지게 된다.

L2에서의 VLAN은 **브로드 캐스트 범위**를 나누는 것이다. 특정 PC가 특정 IP의 MAC를 알려 할 때 L2는 다른 VLAN에 있는 PC의 MAC정보를 제공하지 않는다.

L2에서의 VLAN은 기본적으로 서로 통신할 수 없다. 이를 위해서 L3장비가 존재한다.
L3장비는 스위치이자 동시에 라우터가 필요하다.

사실상 L2에 VLAN이 3개가 존재한다면 L2스위치가 3개 존재한다 보면 되며
거기에 다른 L2장비가 추가로 연결된 L3스위치가 있다면 
L3: VLAN1,2,3,x 의 게이트웨이 역할. L3 밖에서는 모두 같은 IP로 보임.
- L2(1) - VLAN1
- L2(1) - VLAN2
- L2(1) - VLAN3
- L2(2) - VLANx
식으로 연결되었다고 보면 된다.

### 고칠거
대역폭: 속도/용량에 가까움
-> 네트워크 대역

L2는 MAC정보를 제공하지는 않고 그냥 브로드캐스트 범위를 제한하는 역할만 함.

L2스위치는 외부에서 모두 같게 보이는게 아님. 각 게이트웨이 IP를 그대로 사용.
밖에서 같은 IP로 보이는건 보통 **NAT**가 있는 라우터/방화벽을 통해 이루어짐.

# 나의 설명 - DHCP
DHCP는 기본적으로 PC에게 네트워크 설정을 자동으로 주는 역할이다. 
- IP 주소
- 서브넷 마스크
- 기본 게이트웨이 위치
- DNS 서버 주소
- 임대 시간

일반적으로 작은 회사의 경우에는 공유기가 자체적으로 DHCP 서버의 역할을 하며 
큰 회사는 
- Windows Server
- Linux DHCP
- 방화벽
- L3 스위치 
등을 이용한다.

# 질문
## 1. 큰 회사에 DHCP 전용 서버?
보통 큰 회사는 
- Windows Server DHCP
- Linux DHCP
- 전용 Appliance를 사용한다. (DHCP + AD + DNS가 보통 큰 회사, 더 크면 3개 다 분리)

방화벽은 보안 장비이지만 VPN, IPS, DHCP, 방화벽 정책 등 여러 역할을 하는 경우가 꽤 있다.
어짜피 방화벽이 내부망의 게이트웨이 역할을 하기 때문이다.

## 2. L3 스위치는 VLAN간 라우팅을 해주나?
L3에 VLAN1, 2가 있을 때 게이트웨이 A,B간의 통신을 라우팅해주기도 한다.

# 나의 설명 - DNS
DNS는 이름(보통 도메인)을 IP로 바꿔주는 시스템이다.

회사의 방화벽과 라우터 아래에 DNS로 동작하는 서버가 따로 있는데 회사 내부에 존재하는 노드들 (서버 또는 호스트들)의 IP가 이름으로 이 테이블에 등록되게 된다. 
이덕분에 직원들은 보통 
```
\\FILE01\share (FILE01.company.local 로 연결)
http://intranet.company.local
``` 
형태로 자원 및 서버에 접속이 가능하다. (위 방법이 뭐지? 일반 http 요청이랑 다른거같은데)

이전에 말했듯 PC가 DHCP 자동 할당을 통해 네트워크 정보를 받을 때 DNS 서버 주소를 같이 받는다. (IP 모르는 도메인 이름 있으면 이 서버에 요청 보내셈 ㅇㅇ 이라는 뜻)

### 고쳐야할 부분
\\FILE01\share는 Windows 파일 공유 경로로
SMB / CIFS 프로토콜을 통해 FILE이라는 컴퓨터의 share 공유 폴더에 접속하는 방식이다. 
**흐름**
0. \\FILE01\share 작성 (파일 탐색기에)
1. FILE01 이름을 DNS로 IP 조회
2. 해당 IP의 SMB(보통 TCP 445) 서비스로 접속 
3. 권한 확인 
4. 공유 폴더 접근

# 나의 설명 - AD 기본
AD는 Active Directory로 회사 계정, PC, 권한을 관리하는 시스템이다.
예를 들면 
- 직원 계정
- 직원 PC
- 부서 그룹
- 공유폴더 권한
- 로그인 정책
- 비밀번호 정책
등을 한곳에서 관리한다.

이를 통해서 직원들의 계정을 AD에 저장해 놓은 뒤 직원이
`COMPANY\hurwan`, `hurwan@company.local`과 같이 로그인 하게 되면 AD에 저장되어있는 계정과 대조 후 접속이 가능해진다.
(내용이 적어서 추측)
아마 AD는 회사 내 주요 관리 서버로 사용되는 것으로 보인다. Windows Server에서 설정되는게 아닌가 싶다.
이곳에 데이터베이스처럼 [사용자명, 아이디, 비밀번호, 권한 그룹 등]이 저장되며 사용자가 COMPANY\hurwan 또는 hurwan@[회사 도메인] 으로 특정 프로토콜로 요청하게 되면 L2 스위치 -> L3 스위치 -> 회사 라우터 -> ServerCore -> WindowsServer 로 요청이 가서 입력 정보를 기반으로 보안 인증 후 접근 허용 (토큰 등)을 내려주는 것으로 보인다. 이외에도 ServerCore 내부의 공유자원, 서버 제한 등에 대한 다른 장치가 해주지 않는 대부분의 회사 내 관리를 하는 주요 서버에서 이뤄지는 작업 중 하나라고 추측된다.

### 고칠부분
비밀번호는 당연히 해시되어 저장된다. (보통 [계정정보 + 인증용 해시 + 권한정보])

로그인 요청 흐름에 대해서는 
약간틀림: L2 -> L3 -> 회사 라우터 -> ServerCore -> WindowsServer
보통 회사 라우터까지 가지 않고 Access_L2_Switch -> Core_L2_Switch ->  AD서버 로 이루어지며
같은 VLAN이면 직원_PC -> L2_Switch -> AD서버
다른 VLAN이면 직원_PC -> L2_Switch -> L3_Switch -> AD서버_VALN -> AD서버  형태이다.

**ServerCore/WindowsServer은 AD가 아니다.**
ServerCore은 WindowsServer 설치 형태이다. (CLI Windows Server)
AD = Windows Server에 추가하는 형태
