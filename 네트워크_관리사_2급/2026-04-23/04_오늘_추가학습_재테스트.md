## 핵심 지식

## 1. 서브넷 계산

### 제일 먼저 할 것
- 프리픽스 길이(`/26`, `/28` 같은 것)를 보고 `호스트 비트 수`를 확인한다.
- 해당 옥텟의 `블록 크기`를 구한다.

### 블록 크기 구하는 법
- 공식: `256 - 서브넷마스크 값`

예시:
- `/26 = 255.255.255.192` -> `256 - 192 = 64`
- `/28 = 255.255.255.240` -> `256 - 240 = 16`
- `/29 = 255.255.255.248` -> `256 - 248 = 8`

### 자주 나오는 구간
- `/26` -> `0, 64, 128, 192`
- `/28` -> `0, 16, 32, 48, 64, 80 ...`
- `/29` -> `0, 8, 16, 24, 32 ...`

### 무조건 구분할 것
- `네트워크 주소`: 구간의 첫 주소
- `브로드캐스트 주소`: 구간의 마지막 주소
- `사용 가능한 호스트`: 그 사이 주소들

### 제일 많이 틀리는 포인트
- 전체 주소 수와 사용 가능한 호스트 수를 같게 생각하는 것
- 브로드캐스트 주소를 호스트 주소처럼 고르는 것

공식:
- 전체 주소 수: `2^호스트비트`
- 사용 가능 호스트 수: `2^호스트비트 - 2`

예시:
- `/28` -> 호스트 비트 4개 -> 전체 `16`개 -> 사용 가능 `14`개

## 2. 주소 규칙

### 시험에서 규칙으로 보는 것
- 네트워크 주소는 호스트에 할당 불가
- 브로드캐스트 주소는 호스트에 할당 불가

### 시험에서 규칙이 아닌 것
- `.1은 보통 게이트웨이`
- `.254를 게이트웨이로 쓴다`

이건 `관례`일 뿐이다.
문제가 `사용할 수 없는 주소`를 물으면 관례가 아니라 `절대 할당 불가 주소`를 찾아야 한다.

예시:
- `192.168.1.0/24`
- 네트워크 주소: `192.168.1.0`
- 브로드캐스트 주소: `192.168.1.255`
- 사용 가능 호스트: `192.168.1.1 ~ 192.168.1.254`

## 3. 포트 번호

### 최소 암기 세트
- `FTP 20/21`
- `SSH 22`
- `Telnet 23`
- `SMTP 25`
- `DNS 53`
- `HTTP 80`
- `POP3 110`
- `IMAP 143`
- `HTTPS 443`

### 자주 섞는 함정
- `SSH 22`와 `Telnet 23`
- `HTTP 80`과 `HTTPS 443`
- `FTP 20/21`을 하나만 기억하는 실수

## 4. 계층 구분

### 빠르게 정리
- `IP` -> 네트워크 계층
- `ICMP` -> 네트워크 계층 쪽으로 본다
- `ARP` -> 데이터링크/네트워크 경계에서 자주 묶여 나오지만 시험에서는 `주소 변환` 키워드로 기억
- `TCP/UDP` -> 전송 계층
- `HTTP, FTP, SMTP, DNS` -> 응용 계층

### 자주 나오는 문장형 키워드
- `신뢰성, 흐름 제어, 오류 제어` -> TCP 또는 전송 계층
- `비연결형, 빠름, 오버헤드 적음` -> UDP
- `이름 해석` -> DNS
- `IP 자동 할당` -> DHCP
- `통신 가능 여부 확인` -> ping

## 재테스트 1회

이번에는 아까 틀렸던 포인트를 중심으로 다시 묻는다.
답은 `번호 + 한 줄 근거`로 적는다.

1. `10.10.10.70/26`이 속한 네트워크 주소는?
   1. `10.10.10.0`
   2. `10.10.10.64`
   3. `10.10.10.128`
   4. `10.10.10.192`
  정답: 2 (255.255.255.192 -> 64 기준으로 끊어서 사용)

2. `255.255.255.248` 서브넷 마스크를 사용하는 네트워크의 사용 가능한 호스트 수는?
   1. 4
   2. 6
   3. 8
   4. 14
  정답: 32개 (248 = 128 + 64 + 32 + 16 + 8 . 4 + 2 + 1 -> 2^5)

3. `172.16.1.0/28` 네트워크에서 브로드캐스트 주소는?
   1. `172.16.1.14`
   2. `172.16.1.15`
   3. `172.16.1.16`
   4. `172.16.1.31`
  정답: 2 (11110000 에서 2^4 의 최댓값-1 = 145)

4. 다음 중 일반 호스트에 할당할 수 없는 주소는?
   1. `192.168.5.1/24`
   2. `192.168.5.100/24`
   3. `192.168.5.254/24`
   4. `192.168.5.255/24`
  정답: 4 (브로드캐스트 주소라)

5. 다음 중 옳은 포트 연결은?
   1. SSH - 23
   2. Telnet - 22
   3. HTTPS - 443
   4. DNS - 25
   정답: 3 (DNS: 53, 텔넷: 23, ssh: 22)

6. 다음 중 응용 계층 프로토콜만으로 묶인 것은?
   1. TCP, UDP, IP
   2. HTTP, FTP, SMTP
   3. IP, ICMP, ARP
   4. TCP, HTTP, DNS
   정답: tcp/udp: 전송, icmp, ip: 네트워크, http/ftp/smtp: 응용, arp: 데이터링크

7. `연결지향형이고 신뢰성 있는 전송`이라는 설명에 가장 적절한 것은?
   1. UDP
   2. TCP
   3. IP
   4. ARP
   정답: TCP (그냥 존재 자체가 그게 목적이라? 3handshake를 통해?)

8. `ping` 명령과 가장 직접적으로 연결되는 것은?
   1. DHCP
   2. ICMP
   3. FTP
   4. SMTP
   정답: icmp 를 통해 실제로 잘 가는지, request(8) -> reply(0) 으로 해결

9. `192.168.30.33/28`이 속한 네트워크 주소는?
   1. `192.168.30.16`
   2. `192.168.30.32`
   3. `192.168.30.48`
   4. `192.168.30.64`
   정답: 2 (마지막이 11110000 이라 16 단위로 끊어짐)

10. 다음 중 맞는 설명은?
    1. `.1` 주소는 항상 호스트에 할당 불가이다.
    2. 브로드캐스트 주소는 일반 호스트에 할당할 수 있다.
    3. 네트워크 주소와 브로드캐스트 주소는 호스트에 할당할 수 없다.
    4. 게이트웨이 주소는 반드시 `.254`여야 한다.
    정답: 3 (나머지는 전부 바꿀 수 있거나 호스트에게 할당 불가)

## 재테스트 정답

1. `2`
   `/26`은 블록 크기 `64`이므로 `64~127` 구간이다.

2. `2`
   `/29`는 호스트 비트 3개이므로 `2^3 - 2 = 6`.

3. `2`
   `/28`의 블록 크기는 `16`이므로 `0~15` 구간, 마지막 주소 `15`가 브로드캐스트다.

4. `4`
   `/24`에서 `192.168.5.255`는 브로드캐스트 주소다.

5. `3`
   HTTPS의 기본 포트는 `443`이다.

6. `2`
   HTTP, FTP, SMTP는 모두 응용 계층이다.

7. `2`
   연결지향형, 신뢰성, 순서 보장은 TCP의 대표 키워드다.

8. `2`
   ping은 ICMP Echo Request/Reply를 사용한다.

9. `2`
   `/28`은 `16` 단위로 끊기므로 `33`은 `32~47` 구간에 속한다.

10. `3`
    네트워크 주소와 브로드캐스트 주소만이 규칙적으로 할당 불가다.

## 결과 해석

- `9~10개`: 오늘 부족한 부분 보강이 잘 된 상태다. 바로 기출 1회로 넘어가면 된다.
- `7~8개`: 아직 서브넷 계산이 덜 굳었다. 틀린 문제만 다시 계산해 보고 기출로 넘어간다.
- `6개 이하`: 기출 들어가기 전에 서브넷만 10문제 더 푸는 편이 낫다.

### 틀린 문제 분석

#### 2번
- 네 답: `32개`
- 정답: `6개`
- 이유: `255.255.255.248`은 `/29`다.
- `/29`는 호스트 비트가 `3개`이므로 전체 주소 수는 `2^3 = 8`, 사용 가능한 호스트 수는 `8 - 2 = 6`이다.
- 여기서 중요한 건 `248`을 이진수로 바꾸는 것보다 `255.255.255.248 = /29`라는 형태를 빠르게 보는 것이다.
- 분류: `서브넷 계산 실수`

### 맞은 문제 중 체크할 부분

#### 3번
- 계산 방향은 맞다.
- 다만 설명에 `145`처럼 적힌 부분은 단순 오타로 보인다.
- `/28`이면 블록 크기 `16`, `0~15` 구간의 마지막 주소 `15`가 브로드캐스트라는 흐름만 정확히 유지하면 된다.

#### 6번
- 선택지 번호를 명확히 적지 않았지만, 적은 근거상 `2번`을 고른 것으로 판단했다.
- 다음부터는 시험처럼 `번호`를 먼저 적는 습관이 좋다.
