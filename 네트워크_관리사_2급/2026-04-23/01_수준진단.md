# 수준 진단 1회

아래 문제는 `기본 개념 암기 여부`가 아니라 `시험에서 실제로 갈리는 세부 포인트` 위주로 구성했다.

## 푸는 방법
- 각 문항은 답만 쓰지 말고 `왜 그 답인지 한 줄 근거`까지 적는다.
- 확신이 없으면 `찍은 문제` 표시를 남긴다.
- 먼저 풀고, 채점은 맨 아래 `정답 및 해설`을 확인한다.

---

## 문제

1. `192.168.10.130/26`이 속한 네트워크 주소는?
   1. `192.168.10.0`
   2. `192.168.10.64`
   3. `192.168.10.128`
   4. `192.168.10.192`
  정답: 4 (26 -> 8, 8, 8, 2(11000000=192))

2. 다음 중 전송 계층 프로토콜이 아닌 것은?
   1. TCP
   2. UDP
   3. IP
   4. SCTP
  정답: 3 (ip는 네트워크 sctp는 잘 모르겠지만 1,2는 맞고 3번은 확실히 아님)

3. 다음 설명에 해당하는 장비는?
   `MAC 주소를 기반으로 프레임을 전달하며, 충돌 도메인을 분리한다.`
   1. 허브
   2. 리피터
   3. 스위치
   4. 게이트웨이
  정답: 3 (vlan으로 나누면서 작업을 하기도 하고 허브의 상위호환 역할도 함. 뭔가 기능이 더 있는걸로 알고있음)

4. `ping` 명령의 기본적인 동작에 직접적으로 사용되는 프로토콜은?
   1. ARP
   2. ICMP
   3. UDP
   4. SNMP
  정답: 2 (icmp기반으로 udp 보내서 뭐 icmp message나 ttl을 기반으로 작업 하기도 하는게 있는걸로 알고있음. tracroute였나)

5. 다음 중 `잘 알려진 포트(Well-Known Port)` 영역에 속하는 것은?
   1. `0~1023`
   2. `0~2047`
   3. `1024~49151`
   4. `49152~65535`
  정답: 1 (그냥 딱 봐도 80: http, 20,21: ftp 등등 다 세자릿수 안임)

6. OSI 7계층 중 `종단 간 신뢰성 있는 전송, 흐름 제어, 오류 제어`와 가장 관련이 깊은 계층은?
   1. 네트워크 계층
   2. 전송 계층
   3. 세션 계층
   4. 표현 계층
  정답: 2 (대표적으로 tcp의 슬라이딩 윈도우, XON/XOFF)

7. 다음 중 `연결지향형` 서비스는?
   1. UDP
   2. IP
   3. TCP
   4. ICMP
  정답: 3 (3핸드셰이크 syn, syn+ack, ack로 연결, 모든 작업에서 이걸 하는건가?)

8. `255.255.255.240` 서브넷 마스크를 사용하는 네트워크의 호스트 수는?
   1. 14
   2. 16
   3. 30
   4. 62
  정답: 2 (11111111.11111111.11111111.11110000 -> 2^4)

9. 다음 중 `도메인 이름을 IP 주소로 변환`하는 데 가장 직접적인 역할을 하는 것은?
   1. DHCP
   2. DNS
   3. SNMP
   4. NAT
  정답: DNS (그냥 DNS자체의 존재 의의가 이거임. 나머지는 IP제공/변환/UDP오류체크 같은거)

10. IPv4 주소 체계에서 브로드캐스트 주소의 의미로 가장 적절한 것은?
    1. 동일 네트워크 내 모든 호스트에게 전송하기 위한 주소
    2. 특정 라우터 한 대를 지정하는 주소
    3. 원격지 서버의 기본 주소
    4. 네트워크의 첫 번째 사용 가능 호스트 주소
  정답: 1 (말 그대로. 유니캐스트가 1ㄷ1, 멀티가 1:N)

11. 다음 중 `ipconfig /all`로 가장 적절하게 확인할 수 있는 정보는?
    1. 라우터까지의 경로
    2. 로컬 인터페이스의 IP 설정 상세 정보
    3. 도메인 질의 결과
    4. 열린 TCP 세션 목록
  정답: 2 (몰라서 확률로 찍었는데 ipconfig 하면 현재 pc의 설정 나오니까 연결된 ip가 나올거같음)

12. 다음 중 라우터의 주 기능으로 가장 적절한 것은?
    1. 동일 네트워크 내 신호 증폭
    2. 서로 다른 네트워크 간 경로 선택 및 패킷 전달
    3. 프레임 충돌 재생성
    4. 응용 계층 데이터 암호화
  정답: 2 (경로 찾아줌. OSFP, RIP, BFP 였나 이런걸로 Distance기반이든 Vector 기반이든 으로 찾아감)

13. `HTTP`의 기본 포트 번호는?
    1. 20
    2. 21
    3. 23
    4. 80
  정답: 80 (20,21은 FTP, 23은 SSH였나)

14. 다음 중 `전송 계층`과 `응용 계층`의 조합이 옳은 것은?
    1. IP - FTP
    2. TCP - SMTP
    3. ARP - DNS
    4. ICMP - HTTP
  정답: 2 (IP: 네트워크, FTP: 전송계층, TCP: 전송, SMTP: 응용, ARP: 데이터링크(네트워크), DNS: 전송(네트워크), ICMP: 네트워크, HTTP: 응용계층)

15. `192.168.1.0/24` 네트워크에서 일반적인 호스트가 사용할 수 없는 주소는?
    1. `192.168.1.1`
    2. `192.168.1.25`
    3. `192.168.1.254`
    4. `192.168.1.255`
  정답: 1 (보통 1은 게이트웨이임)

---

## 정답 및 해설

1. `3`
   `/26`은 블록 크기 `64`이므로 구간은 `0, 64, 128, 192`이다. `130`은 `128~191` 구간에 속한다.

2. `3`
   IP는 네트워크 계층 프로토콜이다. TCP, UDP, SCTP는 전송 계층이다.

3. `3`
   스위치는 MAC 주소 테이블을 참조해 프레임을 전달하고, 포트별로 충돌 도메인을 분리한다.

4. `2`
   `ping`은 ICMP Echo Request/Reply를 이용한다. ARP는 같은 네트워크에서 MAC 주소를 찾을 때 보조적으로 보일 수 있지만 정답은 ICMP다.

5. `1`
   잘 알려진 포트는 `0~1023`이다. `1024~49151`은 등록 포트, `49152~65535`는 동적/사설 포트다.

6. `2`
   신뢰성, 흐름 제어, 오류 제어는 전형적인 전송 계층 기능이다.

7. `3`
   TCP는 연결지향형, UDP는 비연결형이다. 이 구분은 매우 자주 나온다.

8. `1`
   `255.255.255.240`은 `/28`이고 호스트 비트는 4비트다. 사용 가능한 호스트 수는 `2^4 - 2 = 14`.

9. `2`
   DNS는 이름 해석, DHCP는 IP 자동 할당, NAT는 사설/공인 주소 변환이다.

10. `1`
    브로드캐스트 주소는 해당 네트워크의 모든 호스트에게 동시에 보내기 위한 주소다.

11. `2`
    `ipconfig /all`은 IP, 서브넷 마스크, 게이트웨이, MAC 주소, DHCP 여부 등 인터페이스 상세 정보를 보여준다.

12. `2`
    라우터는 네트워크 계층 장비로, 서로 다른 네트워크 사이에서 패킷을 전달한다.

13. `4`
    `HTTP 80`, `HTTPS 443`, `FTP 20/21`, `Telnet 23`, `SMTP 25`는 기본 암기 영역이다.

14. `2`
    TCP는 전송 계층, SMTP는 응용 계층이다. 나머지는 계층 조합이 맞지 않는다.

15. `4`
    `/24`에서 `192.168.1.255`는 브로드캐스트 주소다. 일반 호스트에 할당할 수 없다.

---

## 결과 해석

- `13~15개`: 기본 개념은 강하다. 이제 보기 함정과 서브넷 속도를 올려야 한다.
- `10~12개`: 실전권이다. 포트 번호와 계층 구분을 더 정교하게 다듬어야 한다.
- `7~9개`: 개념이 연결되지 않는 부분이 있다. 틀린 유형을 바로 분류해야 한다.
- `0~6개`: 개념을 다시 잡는 편이 빠르다. 특히 IP 주소 계산과 프로토콜 구분부터 재정리한다.

## 이번 풀이 해석

### 총점
- `12/15`

### 현재 수준 평가
- 기본 개념은 잡혀 있다.
- 실전 기준으로는 `개념 부족`보다 `세부 규칙에서 실수하는 단계`에 가깝다.
- 특히 `서브넷`, `사용 가능 호스트 수`, `브로드캐스트 주소` 같은 주소 계산형 문제에서 점수가 갈릴 가능성이 높다.

### 틀린 문제 분석

#### 1번
- 네 답: `4번`
- 정답: `3번`
- 이유: `/26`은 블록 크기가 `64`이므로 구간은 `0, 64, 128, 192`이다.
- `192.168.10.130`은 `128~191` 구간에 속하므로 네트워크 주소는 `192.168.10.128`이다.
- 분류: `서브넷 계산 실수`

#### 8번
- 네 답: `2번`
- 정답: `1번`
- 이유: `255.255.255.240`은 `/28`이고 호스트 비트는 4비트다.
- 전체 주소 수는 `2^4 = 16`이지만, 사용 가능한 호스트 수는 네트워크 주소와 브로드캐스트 주소를 제외해서 `14`다.
- 분류: `서브넷 계산 실수`

#### 15번
- 네 답: `1번`
- 정답: `4번`
- 이유: `192.168.1.1`은 게이트웨이로 자주 쓰일 수는 있지만 일반 호스트가 절대 사용할 수 없는 주소는 아니다.
- 반면 `192.168.1.255`는 `/24` 네트워크의 브로드캐스트 주소이므로 호스트에 할당할 수 없다.
- 시험은 `관례`가 아니라 `주소 규칙`을 묻는다.
- 분류: `주소 규칙 혼동`

### 맞혔지만 보완이 필요한 부분

#### 13번 관련
- `23번 포트`는 `SSH`가 아니라 `Telnet`이다.
- `SSH`는 `22번 포트`다.
- 이번 문제는 맞혔지만 포트 번호 문제에서 실제로 틀릴 수 있는 부분이다.

#### 14번 관련
- `FTP`, `DNS`, `SMTP`, `HTTP`는 모두 `응용 계층`으로 정리하는 것이 안전하다.
- `IP`는 네트워크 계층, `TCP/UDP`는 전송 계층이라는 축과 함께 묶어서 외워야 한다.

#### 4번 관련
- `ping`의 핵심 프로토콜은 `ICMP`다.
- ARP, TTL 같은 주변 개념을 같이 떠올릴 수는 있지만 정답 판단의 중심은 `ICMP`여야 한다.
