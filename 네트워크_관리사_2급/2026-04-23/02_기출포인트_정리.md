# 기출 포인트 정리

이 문서는 `단순 개념 암기`보다 `시험에서 보기로 헷갈리게 섞는 지점`에 초점을 둔다.

## 1. OSI 7계층과 TCP/IP

### 반드시 구분할 것
- OSI 7계층: 물리, 데이터링크, 네트워크, 전송, 세션, 표현, 응용
- TCP/IP 4계층: 네트워크 액세스, 인터넷, 전송, 응용

### 시험 포인트
- `IP`는 네트워크 계층
- `TCP/UDP`는 전송 계층
- `HTTP, FTP, SMTP, DNS`는 응용 계층
- `MAC 주소`는 데이터링크 계층과 연결
- `라우팅`은 네트워크 계층 핵심 기능
- `신뢰성, 흐름 제어, 오류 제어`는 전송 계층 핵심 키워드

### 자주 섞는 함정
- `세션 계층`과 `전송 계층`을 섞는다.
- `IP`를 전송 계층으로 착각하게 만든다.
- `DNS`를 네트워크 계층처럼 보이게 낸다.

## 2. 장비 구분

### 허브
- 물리 계층 장비
- 들어온 신호를 모든 포트로 전달
- 충돌 도메인 분리 못함
- 보안성과 효율이 낮다

### 스위치
- 데이터링크 계층 장비
- MAC 주소 기반 전달
- 충돌 도메인 분리
- 같은 네트워크 내 통신에 강함

### 라우터
- 네트워크 계층 장비
- IP 주소 기반 경로 선택
- 서로 다른 네트워크 연결
- 브로드캐스트 도메인 분리

### 자주 나오는 비교
- `충돌 도메인 분리`: 스위치
- `브로드캐스트 도메인 분리`: 라우터
- `신호 재생/증폭`: 리피터

## 3. IPv4 주소와 서브넷

### 무조건 익혀야 할 공식
- 호스트 수: `2^h - 2`
- 블록 크기: `256 - 서브넷마스크 옥텟값`

### 자주 쓰는 프리픽스
- `/24 = 255.255.255.0` -> 호스트 `254`
- `/25 = 255.255.255.128` -> 호스트 `126`
- `/26 = 255.255.255.192` -> 호스트 `62`
- `/27 = 255.255.255.224` -> 호스트 `30`
- `/28 = 255.255.255.240` -> 호스트 `14`
- `/29 = 255.255.255.248` -> 호스트 `6`
- `/30 = 255.255.255.252` -> 호스트 `2`

### 꼭 구분할 것
- 네트워크 주소: 해당 구간의 첫 주소
- 브로드캐스트 주소: 해당 구간의 마지막 주소
- 사용 가능 호스트: 네트워크/브로드캐스트 제외

### 매우 자주 틀리는 포인트
- `/26`에서 블록 크기 `64`를 놓침
- `사용 가능한 주소 수`와 `전체 주소 수`를 혼동
- `192.168.1.255` 같은 브로드캐스트 주소를 호스트 주소처럼 고름

## 4. 프로토콜과 포트 번호

### 기본 암기
- `FTP 20/21`
- `SSH 22`
- `Telnet 23`
- `SMTP 25`
- `DNS 53`
- `HTTP 80`
- `POP3 110`
- `NNTP 119`
- `IMAP 143`
- `HTTPS 443`

### 함정 포인트
- FTP는 `20, 21` 둘 다 기억해야 한다.
- DNS는 이름 해석 기능과 `53번 포트`를 함께 묻는다.
- Telnet과 SSH는 둘 다 원격 접속이지만 보안성에서 차이가 있다.

## 5. TCP와 UDP

### TCP
- 연결지향형
- 신뢰성 보장
- 흐름 제어, 오류 제어
- 순서 보장

### UDP
- 비연결형
- 속도 우선
- 신뢰성 보장 약함
- 실시간 서비스에 사용

### 시험에서 묻는 식
- `신뢰성 있다` -> TCP
- `오버헤드 적다` -> UDP
- `연결 설정 과정이 없다` -> UDP
- `순서 보장` -> TCP

## 6. 주요 명령어

### ipconfig
- 로컬 네트워크 설정 확인
- `ipconfig /all`은 상세 정보 출력

### ping
- ICMP 기반 연결 확인
- 상대가 응답 가능한지 확인

### tracert
- 목적지까지 거치는 경로 추적
- 중간 홉 확인

### nslookup
- DNS 질의 결과 확인
- 도메인과 IP 매핑 점검

### 자주 나오는 연결
- `IP 설정 상세 확인` -> `ipconfig /all`
- `통신 가능 여부 확인` -> `ping`
- `경로 추적` -> `tracert`
- `DNS 확인` -> `nslookup`

## 7. 자주 나오는 용어

### ARP
- 같은 네트워크에서 IP를 MAC 주소로 변환

### NAT
- 사설 IP를 공인 IP로 변환
- 여러 내부 호스트가 외부 통신할 때 자주 연결

### DHCP
- IP 주소 자동 할당

### DNS
- 도메인 이름을 IP 주소로 변환

### ICMP
- 오류 보고, 진단
- `ping`과 강하게 연결

## 8. 공부할 때 기준

문제를 풀 때는 아래 4개를 동시에 확인해야 한다.

- `무슨 계층인가`
- `무슨 주소를 쓰는가`
- `무슨 기능을 하는가`
- `무슨 숫자와 연결되는가`

예를 들어 DNS를 보면:
- 계층: 응용 계층
- 기능: 이름 해석
- 포트: 53
- 헷갈리는 대상: DHCP, ARP
