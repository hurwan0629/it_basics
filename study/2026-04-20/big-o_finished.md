# Big-O 표기법

- 날짜: 2026-04-20
- 분류: 알고리즘 / 자료구조
- 체크리스트 항목: Big-O 표기법을 기본 문제에 적용할 수 있다
- 오늘 이걸 공부하는 이유: 알고리즘을 볼 때 무엇이 더 비효율적인지 판단하는 기준이 필요하다.

## 한 줄 정의

Big-O는 입력 크기가 커질 때 알고리즘의 실행 시간이나 사용 공간이 얼마나 늘어나는지를 단순화해서 나타내는 표기법이다.

## 핵심 개념

- 상수, 낮은 차수 항은 보통 무시하고 가장 크게 증가하는 항만 본다
- 시간 복잡도와 공간 복잡도 모두 Big-O로 표현할 수 있다
- 입력이 커질수록 어떤 알고리즘이 더 불리해지는지 비교할 때 유용하다
- 정확한 실행 시간 자체보다 증가 경향을 보는 개념이다

## 동작 흐름 / 원리

- 반복문이 1개면 보통 `O(n)`인지 먼저 본다
- 중첩 반복문이 2개면 보통 `O(n^2)`인지 본다
- 입력을 반씩 줄이면 `O(log n)` 가능성을 본다
- 여러 단계가 이어지면 가장 큰 비용을 기준으로 정리한다

## 예시

- 배열 전체를 한 번 순회해서 최댓값 찾기: `O(n)`
- 정렬된 배열에서 이진 탐색하기: `O(log n)`
- 이중 반복문으로 모든 쌍 비교하기: `O(n^2)`
- 배열 첫 번째 값 바로 꺼내기: `O(1)`

## 비슷한 개념과 차이

- 실제 실행 시간: 컴퓨터 성능, 언어, 구현 방식의 영향을 받는다
- Big-O: 입력이 커질 때 증가 추세를 본다

- 시간 복잡도: 연산 횟수 증가를 본다
- 공간 복잡도: 추가 메모리 사용 증가를 본다

## 헷갈린 점

- `O(2n)`을 왜 `O(n)`으로 줄여서 보는지 처음에는 직관이 약하다
- `O(log n)`이 나오는 상황을 더 많이 봐야 감이 잡힌다
- 최선, 평균, 최악의 경우를 언제 구분해서 봐야 하는지 추가 학습이 필요하다

## 확인 문제

- [ ] 단일 반복문으로 배열 합을 구하는 코드는 왜 `O(n)`인가?
- [ ] 이중 반복문으로 모든 원소 쌍을 비교하면 왜 `O(n^2)`인가?

## 3줄 요약

1. Big-O는 정확한 시간보다 입력 증가에 따른 증가 폭을 보는 표기다.
2. 가장 크게 증가하는 항 중심으로 시간 복잡도를 정리한다.
3. 반복문 수, 입력 감소 방식, 추가 메모리 사용을 함께 보는 습관이 중요하다.
