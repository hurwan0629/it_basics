## Round 1 테스트

### Q1. 동시성이 무엇인지 한두 문장으로 설명하라

답: 동시성은 cpu의 하나의 코어에서 동시에 2개 이상의 작업이 물리적으로 실행이 불가능 한 것 (스케줄러가 실행할 작업을 스케줄링하여 빠르게 작업을 바꾸며 작업하는것)

### Q2. 병렬성이 무엇인지 한두 문장으로 설명하라

답: 병렬성은 2개 이상의 cpu코어가 동시에 다른 작업을 수행하는 것

### Q3. 동시성과 병렬성의 가장 큰 차이를 적어라

답: 동시성은 실제로 다중화가 아닌데 병렬성은 실제로 다중화 작업을 함

### Q4. 싱글코어 CPU에서 여러 작업을 번갈아 처리하는 상황은 왜 동시성이라고 부를 수 있는지 적어라

답: 동시에 같은 작업을 하지 않기 때문에

### Q5. 동시성/병렬성과 동기/비동기가 왜 같은 말이 아닌지 적어라

답: 동시성/병렬성은 그냥 cpu가 어떻게 프로세스 및 커널 등의 연산을 수행하는지를 나타낸 것이고 동기/비동기는 작업을 기다리냐 마느냐이기 때문에 다른 주제를 말한다.

## Round 1 피드백

- Q1: 방향은 맞지만 문장이 조금 꼬였다. 동시성의 핵심은 `하나의 순간에 하나만 실행되더라도 여러 작업을 겹쳐 관리하며 번갈아 진전시키는 것`이다. 꼭 `하나의 코어`에만 묶을 필요는 없다. 멀티코어 환경에서도 동시성 개념은 여전히 성립한다.
- Q2: 좋다. 병렬성을 `실제로 동시에 실행되는 것`으로 잡은 점이 핵심이다. 다만 CPU 코어뿐 아니라 여러 실행 자원에서 동시에 수행되는 경우로 넓게 이해해도 된다.
- Q3: `다중화`라는 표현보다 `번갈아 처리하느냐, 실제로 동시에 실행되느냐`로 말하는 편이 더 정확하다.
- Q4: 여기서는 이유가 부족하다. 핵심은 `싱글코어여도 CPU가 아주 빠르게 작업을 전환하면서 각 작업이 조금씩 진행되기 때문에, 사용자 입장에서는 여러 작업이 겹쳐 진행되는 것처럼 보인다`는 점이다.
- Q5: 방향은 맞다. 다만 `커널 등의 연산`보다는 `여러 작업을 처리하는 방식`이라고 더 일반적으로 표현하는 편이 안전하다. 동시성/병렬성은 작업 처리 구조, 동기/비동기는 결과를 기다리는 방식이다.

## 보강 정리

### 한 줄 정의

- 동시성: 여러 작업을 겹쳐 관리하며, 번갈아 조금씩 진행하는 처리 방식
- 병렬성: 여러 작업이 실제로 같은 시각에 함께 실행되는 처리 방식

### 핵심 포인트

- 동시성은 `작업을 어떻게 겹쳐 다루는가`에 가깝다
- 병렬성은 `실제로 동시에 실행되는가`에 가깝다
- 싱글코어에서도 동시성은 가능하다
- 병렬성은 보통 여러 코어 또는 여러 실행 자원이 있어야 의미가 있다
- 동시성은 구조와 스케줄링 관점이고, 병렬성은 실행 자원 관점이 강하다

### 예시

- 동시성: 한 명의 요리사가 라면 물 올리기, 계란 삶기, 접시 준비를 번갈아 처리한다
- 병렬성: 두 명의 요리사가 동시에 각자 다른 요리를 진행한다

### 헷갈리기 쉬운 비교

- 동시성/병렬성: 여러 작업을 처리하는 방식의 차이
- 동기/비동기: 결과를 기다리는 방식의 차이

- 동시성이라고 해서 반드시 여러 코어가 필요한 것은 아니다
- 병렬성은 동시성을 포함할 수 있지만, 둘은 완전히 같은 말은 아니다

## Round 2 재테스트

### Q1. 동시성과 병렬성의 차이를 한 문장으로 적어라

답: 동시성은 한번에 하나의 작업만 가능하지만 병렬성은 여러 cpu코어를 통해 동시에 2개 이상의 작업이 가능하다.

### Q2. 싱글코어에서도 동시성이 가능한 이유를 짧게 적어라

답: 하나의 cpu가 여러 작업을 번갈아가며 수행할 수 있기 때문이다.

### Q3. 멀티코어에서 병렬성이 가능한 이유를 짧게 적어라

답: 각기 다른 코어가 각각의 작업을 동시에 수행 가능하기 떄문이다.

### Q4. 동시성/병렬성과 동기/비동기의 차이를 한 문장으로 적어라

답: 동시성/병렬성은 작업을 동시에 할수 있냐 없냐, 동기/비동기는 대기해야하냐 아니냐에 대한 이야기이다. 모두 여러 작업을 동시에 처리가 가능하냐 라는 느낌에 대한 이야기를 하지만 실제는 완전 다르다.

## 최종 정리

- 동시성은 여러 작업을 한 번에 하나씩 번갈아 처리하면서 전체적으로 겹쳐 진행시키는 방식이다
- 병렬성은 여러 코어나 실행 자원을 이용해 여러 작업이 실제로 같은 시각에 함께 실행되는 것이다
- 동기/비동기는 기다리는 방식의 차이이고, 동시성/병렬성은 여러 작업을 처리하는 구조의 차이이다
