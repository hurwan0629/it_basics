# Study Writing Guide

이 문서는 학습 기록을 남길 때의 지침서다.

## 목적

이 공간의 목적은 자료를 많이 모으는 것이 아니라, 개념을 내 말로 설명할 수 있게 만드는 것이다.

## 기본 원칙

- 정의를 외우기보다 흐름을 이해한다
- 짧게 쓰되, 내가 다시 읽었을 때 이해되어야 한다
- 복붙보다 직접 문장으로 다시 쓴다
- 예시를 반드시 넣는다
- 비슷한 개념과의 차이를 같이 적는다
- 끝날 때 모르는 점을 남긴다

## 하루 학습 방식

하루 학습은 아래 순서로 진행한다.

1. `checklist.md`에서 오늘 처리할 항목을 하나 고른다
2. 그 항목 문장을 파일 상단에 그대로 적는다
3. 왜 이 항목이 중요한지 한 줄로 적는다
4. 자료를 보고 핵심 개념을 정리한다
5. 예시를 붙인다
6. 비슷한 개념과 비교한다
7. 스스로 질문을 만든다
8. 마지막에 3줄 요약을 적는다

## 파일 작성 규칙

- 위치: `study/[날짜]/[주제].md`
- 날짜 폴더 형식: `YYYY-MM-DD`
- 한 파일에는 체크리스트 항목 하나만 작성한다
- 파일 맨 위에는 주제와 날짜를 적는다
- 가능하면 파일명도 체크리스트 항목 기준으로 짓는다
- 통과 판정이 나면 파일명을 `[주제]_finished.md`로 바꾼다
- 섹션 순서는 가능한 한 유지한다
- 완벽하게 쓰려 하지 말고, 이해한 만큼 명확하게 쓴다

## 공통 통과 기준

모든 주제는 아래 5개 기준으로 평가한다.

- 정의: 핵심 개념을 자기 말로 설명할 수 있다
- 원리: 왜 그렇게 동작하는지 이유를 설명할 수 있다
- 구분: 비슷한 개념과의 차이를 설명할 수 있다
- 적용: 예시, 사용 상황, 간단한 문제에 연결할 수 있다
- 안정성: 재테스트에서 방향이 흔들리지 않는다

판정 규칙은 아래와 같다.

- 통과: 5개 중 4개 이상 충족하고, 핵심 개념 오류가 없다
- 보류: 핵심 개념 방향이 틀리거나, 재테스트에서도 반복해서 흔들린다
- 부분 통과: 전체 감은 맞지만 예시, 비교, 이유 설명 중 하나가 약하다

추가 원칙:

- 문장을 길게 잘 쓰는 것은 요구하지 않는다
- 짧아도 핵심이 정확하면 통과 가능하다
- 말투보다 개념 방향과 이유 설명이 더 중요하다
- 용어를 외우는 것보다 자기 말로 설명 가능한지를 우선한다

## 추천 템플릿

아래 템플릿을 복사해서 사용한다.

```md
# 주제명

- 날짜:
- 분류:
- 체크리스트 항목:
- 오늘 이걸 공부하는 이유:

## 한 줄 정의

## 핵심 개념

## 동작 흐름 / 원리

## 예시

## 비슷한 개념과 차이

## 헷갈린 점

## 확인 문제
- [ ] 
- [ ] 

## 3줄 요약
1.
2.
3.
```

## 좋은 학습 기록의 기준

좋은 기록은 아래 조건을 만족한다.

- 나중에 다시 봐도 바로 이해된다
- "왜?"에 대한 답이 들어 있다
- 단순 정의가 아니라 연결 관계가 보인다
- 예시와 비교가 있다
- 내가 모르는 부분이 숨겨져 있지 않다

## 피해야 할 방식

- 책 문장을 그대로 옮기기
- 정의만 쓰고 끝내기
- 예시 없이 추상적으로만 적기
- 모르는 부분을 비워두고 넘어가기
- 한 파일에 여러 주제를 섞어 쓰기

## 복습 규칙

- 공부가 끝난 뒤 `checklist.md`에서 관련 항목을 체크한다
- 하루가 끝나면 오늘 쓴 파일들을 5분 안에 다시 훑는다
- 일주일에 한 번은 헷갈린 점만 따로 모아서 다시 본다
- 설명이 안 되는 주제는 다시 새 파일로 정리한다

## 추천 시작 방식

처음에는 아래처럼 진행하는 편이 안정적이다.

- 큰 주제가 아니라 체크리스트 한 항목만 잡는다
- 한 번에 20분에서 40분 안에 끝낼 수 있는 분량으로 쓴다
- 한 파일을 다 쓴 뒤 바로 체크 여부를 판단한다
- 아직 설명이 안 되면 체크하지 말고 `헷갈린 점`을 늘린다

## 기본 진행 방식

특별한 요청이 없으면 아래 순서로 진행한다.

1. `checklist.md`에서 아직 안 끝난 항목 하나를 고른다
2. `study/[날짜]/[주제].md` 파일을 만든다
3. Round 1 테스트를 만든다
4. 사용자가 답을 적는다
5. 부족한 부분만 피드백한다
6. Round 2 또는 Round 3 재테스트를 만든다
7. 통과 기준에 따라 판정한다
8. 통과면 체크리스트를 체크하고 파일명을 `_finished.md`로 바꾼다

## 다음번 시작 규칙

다음에 사용자가 아래처럼 느슨하게 말해도 같은 방식으로 자동 진행한다.

- `공부하자`
- `아무거나 하자`
- `다음 거 하자`
- `이어하자`
- `오늘은 가볍게 하자`

이런 경우 기본 동작은 아래와 같다.

- 아직 완료되지 않은 체크리스트 항목 중 부담이 적은 것부터 고른다
- 같은 큰 주제를 1~2개 더 이어가되, 너무 무겁지 않게 잡는다
- 테스트 -> 답변 -> 피드백 -> 재테스트 -> 통과 판정 흐름을 유지한다
- 별도 지시가 없으면 이전 학습 맥락을 이어서 비슷한 난이도로 낸다

기본 우선순위:

1. 지금 공부 중인 큰 주제의 미완료 항목
2. 바로 이전 항목과 연결되는 쉬운 항목
3. 한 번에 끝내기 쉬운 항목
