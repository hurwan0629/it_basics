# 연결 리스트

- 날짜: 2026-04-21
- 분류: 알고리즘 / 자료구조
- 체크리스트 항목: 연결 리스트의 특징을 설명할 수 있다
- 체크리스트 항목: 연결 리스트에서 접근, 탐색, 삽입, 삭제의 시간 복잡도를 설명할 수 있다
- 학습 방식: 테스트 -> 내 답변 -> 보강 설명 -> 재테스트 -> 통과 판단

## 목표

- 연결 리스트가 어떤 구조인지 자기 말로 설명할 수 있다
- 배열과 비교해서 차이를 설명할 수 있다
- 접근, 탐색, 삽입, 삭제의 복잡도를 말할 수 있다
- 어떤 상황에서 연결 리스트가 유리한지 예시를 들 수 있다

## 진행 규칙

- 먼저 Round 1 테스트를 푼다
- 답은 각 문제 아래에 직접 적는다
- 부족한 부분만 보강 정리를 참고한다
- Round 2에서 핵심이 바로 나오면 통과로 본다

## Round 1 테스트

### Q1. 연결 리스트가 무엇인지 한두 문장으로 설명하라

답: prev, next, value 가 들어있는 노드들의 연결을 통해서 이루어진 형태의 리스트

### Q2. 배열과 연결 리스트의 가장 큰 구조적 차이를 설명하라

답: 배열은 연속된 메모리 공간을 사용하고 연결리스트는 따로 분리된 여러 메모리 공간을 논리적으로 연결한 형태이다.

### Q3. 연결 리스트에서 인덱스로 바로 접근하기 어려운 이유를 적어라

답: 인덱스를 통해 즉시 인덱스에 대한 저장 위치를 알 수 없기 때문에 처음부터 노드를 타고 올라가는 방향이 필요하다.

### Q4. 연결 리스트에서 중간 삽입이나 삭제가 유리할 수 있는 이유를 적어라

답: 그냥 특정 노드 삭제 후 연결 주소 (prev, next)만 수정하면 되기 때문이다. 삽입 시에도 동일하다.

### Q5. 연결 리스트의 접근, 탐색, 삽입, 삭제 시간 복잡도를 적어라

답: 접근 - O(n), 탐색 - O(n), 삽입 - O(1), 삭제 - O(1) (삽입/삭제는 넣는 속도가 그렇고 탐색/접근 비용은 생각 안함)

## Round 1 피드백

- Q1: 방향은 맞다. 다만 `prev, next`가 항상 있는 것은 아니고, 그건 `이중 연결 리스트`에 더 가깝다. 일반적으로는 `데이터와 다음 노드 참조를 가진 노드들이 연결된 구조`라고 적으면 더 정확하다.
- Q2: 정확하다. 배열은 연속 메모리, 연결 리스트는 흩어진 노드를 참조로 잇는다.
- Q3: 좋다. 인덱스로 주소를 바로 계산할 수 없어서 앞에서부터 따라가야 한다는 점이 핵심이다.
- Q4: 정확하다. 위치를 이미 알고 있으면 연결만 바꾸면 되는 점을 잘 짚었다.
- Q5: 잘 적었다. 다만 `삽입/삭제 O(1)`은 "위치를 이미 알고 있을 때"라는 조건을 항상 같이 붙이는 게 안전하다.

## 보강 정리

### 한 줄 정의

연결 리스트는 각 노드가 데이터와 다음 노드에 대한 참조를 가지고 이어지는 자료구조다.

### 핵심 포인트

- 연속된 메모리 공간에 저장되지 않아도 된다
- 각 원소는 보통 `데이터 + 다음 노드 주소` 형태로 연결된다
- 맨 앞부터 차례대로 따라가야 해서 접근은 느리다
- 위치를 이미 알고 있으면 삽입과 삭제는 유리할 수 있다

### 연산 복잡도

- 접근: `O(n)`
- 탐색: `O(n)`
- 삽입: 위치를 이미 알고 있으면 `O(1)`, 찾는 과정까지 포함하면 `O(n)`
- 삭제: 위치를 이미 알고 있으면 `O(1)`, 찾는 과정까지 포함하면 `O(n)`

### 배열과 비교

- 배열: 연속된 메모리, 인덱스 접근 빠름
- 연결 리스트: 노드 연결 구조, 인덱스 접근 느림
- 배열: 중간 삽입/삭제 시 원소 이동 필요
- 연결 리스트: 연결만 바꾸면 삽입/삭제 가능

### 대표 예시

- 삽입과 삭제가 자주 일어나는 순서 목록
- 음악 재생 목록의 다음 곡 연결
- 해시 충돌 해결 방식 중 체이닝

## Round 2 재테스트

### Q1. 연결 리스트의 가장 큰 특징을 한 문장으로 적어라

답: 노드들의 연결로 이루어진 리스트

### Q2. 연결 리스트의 접근이 `O(n)`인 이유를 적어라

답: 첫 노드부터 하나씩 타고 올라가야해서

### Q3. 연결 리스트에서 삽입/삭제가 유리한 상황을 한 가지 적어라

답: 삽입할 노드의 prev 또는 next 노드를 알고 있을 때 (주소를 가지고 있을 때)

### Q4. 배열보다 연결 리스트가 더 어울리는 상황을 한 문장으로 적어라

답: 삽입과 삭제가 자주 일어나는 정렬되있어야하는 순번. 공간 지역성이 높을 때

## Round 2 피드백

- Q1: 맞다. 여기에 `각 노드가 참조로 이어진다`까지 붙으면 더 완전하다.
- Q2: 정확하다. 연결 리스트 접근이 느린 이유를 잘 설명했다.
- Q3: 정확하다. 위치를 알고 있을 때 삽입/삭제가 유리하다는 핵심을 잡았다.
- Q4: 앞부분은 괜찮지만 `공간 지역성이 높을 때`는 연결 리스트보다 배열 쪽에 더 가까운 표현이라 빼는 게 맞다. 연결 리스트가 더 어울리는 상황은 `중간 삽입/삭제가 자주 일어나고, 인덱스 접근보다 구조 변경이 더 중요한 경우`다.

## Round 3 최종 확인

### Q1. 연결 리스트를 일반형 기준으로 한 문장으로 설명하라

답: 다음 노드의 주소를 가지고 있는 노드들로 이루어진 리스트

### Q2. 연결 리스트의 삽입/삭제가 `O(1)`인 조건을 한 문장으로 적어라

답: 삽입할 노드의 prev 노드를 알고 있을 때 또는 삭제할 노드의 주소를 알고있을 때

### Q3. 배열보다 연결 리스트가 더 어울리는 상황을 한 문장으로 적어라

답: 추가/삭제할 값의 노드들을 캐싱해 두는 경우가 많고 삽입/삭제가 많은 자료구조를 사용하는 경우.

## Round 3 피드백

- Q1: 정확하다. 일반 연결 리스트 기준 설명으로 충분하다.
- Q2: 핵심은 맞다. 더 일반적으로는 `삽입/삭제할 위치를 이미 알고 있을 때`라고 정리하면 된다.
- Q3: 방향은 맞다. 중간 삽입/삭제가 자주 일어나고 인덱스 접근이 덜 중요한 상황이라고 이해하면 된다.

## 최종 정리

- 연결 리스트는 각 노드가 다음 노드를 가리키며 이어지는 자료구조다
